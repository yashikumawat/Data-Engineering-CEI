# Superstore Sales Analysis Using SQL

## Project Overview

This project focuses on performing customer and sales analysis using the Superstore dataset. The objective was to transform raw transactional data into a structured database and extract meaningful business insights through SQL queries. The dataset was normalized into separate tables representing customers, orders, and products to improve data organization and reduce redundancy.

The project emphasizes customer behavior analysis, sales performance evaluation, order tracking, and customer ranking by applying advanced SQL concepts such as joins, aggregate functions, subqueries, Common Table Expressions (CTEs), and window functions. These techniques help answer important business questions related to revenue generation, customer contribution, and sales trends.

## Database Structure

The original dataset was divided into three related tables:

### Customers Table

Contains customer-related information such as customer identifiers, names, segments, regions, and location details. This table serves as the primary source for customer analysis.

### Orders Table

Stores order and shipping information including order dates, shipping dates, shipping modes, customer identifiers, and product identifiers. It acts as a bridge connecting customers and products.

### Products Table

Contains product information along with sales metrics such as sales amount, quantity, discount, profit, category, and sub-category. This table is the foundation for all sales and profitability analysis.

## Tasks Performed

### Data Normalization and Table Creation

The first step involved creating separate tables from the raw dataset to establish a normalized database structure. This approach minimizes data duplication and improves query efficiency.

### Subquery-Based Analysis

Several business problems were solved using nested and correlated subqueries:

* Identified orders whose sales were greater than the average sales value.
* Determined the highest sales value associated with each customer.
* Calculated customer-wise total sales using correlated subqueries.
* Found customers whose total sales exceeded the average customer sales.
* Retrieved unique customer records based on specific order conditions.

Subqueries enabled dynamic calculations where the output of one query was used as input for another query, making it possible to perform comparative and hierarchical analysis.

### Common Table Expressions (CTEs)

CTEs were extensively used to simplify complex analytical queries and improve readability.

Key applications included:

* Calculating total sales generated by each customer.
* Identifying customers with above-average sales.
* Determining the highest order value for every customer.
* Creating reusable intermediate result sets for reporting purposes.
* Generating customer performance summaries without creating temporary tables.

CTEs helped break large queries into logical sections, making them easier to understand, maintain, and debug.

### Window Function Analysis

Window functions were used to perform advanced analytical operations while preserving individual records.

The project utilized:

#### ROW_NUMBER()

Assigned sequential numbers to orders within each customer group using `PARTITION BY Customer_ID`. This helped track the order sequence for every customer.

#### RANK()

Ranked customers based on their total sales in descending order. Customers with equal sales received the same rank, and rank gaps were maintained.

#### DENSE_RANK()

Used for customer ranking without gaps between ranks, providing a compact ranking system for performance evaluation.

Window functions allowed analytical calculations to be performed across groups of rows without collapsing the dataset through aggregation.

### Customer Performance Analysis

Customer-focused reports were generated to evaluate business performance:

* Total sales generated by each customer.
* Highest order value for every customer.
* Customers contributing above-average revenue.
* Top-performing customers based on total sales.
* Ranking of customers according to sales contribution.

These analyses provide valuable insights into customer value and purchasing behavior.

### Final Sales Report

A consolidated report was created using JOINs, CTEs, and window functions. The report included:

* Customer Name
* Total Sales
* Customer Rank

This report serves as a complete customer performance dashboard and demonstrates the integration of multiple advanced SQL concepts into a single analytical solution.
